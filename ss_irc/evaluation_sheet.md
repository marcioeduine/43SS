<h3 class="font-weight-bold mb-1"><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">Introdução</font></font></h3>
<p><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">
Por favor, siga as seguintes regras: </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Mantenha-se educado, cortês, respeitoso e construtivo durante todo o </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  processo de avaliação. O bem-estar da comunidade depende disso. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Identifique as possíveis disfunções no projeto do aluno ou </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  grupo cujo trabalho está sendo avaliado. Dedique um tempo para discutir e debater os </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  problemas que possam ter sido identificados. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Você deve considerar que pode haver algumas diferenças na forma como seus colegas </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  interpretaram as instruções do projeto e o escopo de suas </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  funcionalidades. Mantenha sempre a mente aberta e avalie-os da forma mais honesta </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  possível. A pedagogia só é útil se a avaliação por pares for feita com seriedade.</font></font><br>
</p>

<h3 class="font-weight-bold mb-1"><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">Diretrizes</font></font></h3>
<p><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">
- Avalie apenas os trabalhos submetidos no repositório Git do </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  aluno ou grupo avaliado. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Verifique novamente se o repositório Git pertence ao(s) aluno(s). Certifique-se de que </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  o projeto é o esperado. Verifique também se o comando 'git clone' foi usado em um </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  diretório vazio. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Verifique cuidadosamente se nenhum aliase malicioso foi usado para enganá-lo e fazer com que você </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  avalie algo que não corresponde ao conteúdo do repositório oficial. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Para evitar surpresas, e se aplicável, revise em conjunto quaisquer scripts usados </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  ​​para facilitar a avaliação (como scripts de teste ou automação). </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Se você não concluiu a tarefa que irá avaliar, </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  leia todo o conteúdo antes de iniciar o processo de avaliação. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Use as opções disponíveis para reportar um repositório vazio, um programa que não funciona </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  , um erro de norma, plágio, etc. </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  Nesses casos, o processo de avaliação termina e a nota final é 0, </font></font><br><font dir="auto" No entanto, exceto em casos de plágio, os alunos são </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  fortemente encorajados a revisar o trabalho submetido em conjunto </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  para identificar quaisquer erros que não devem ser repetidos no futuro. </font></font><br><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">- Lembre-se que, durante a defesa, não pode haver falhas de segmentação (segfault), nem qualquer outra </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  interrupção inesperada, prematura, descontrolada ou inesperada do </font></font><br><font dir="auto" style="vertical-align: inherit;"><font dir="auto" style="vertical-align: inherit;">  programa; caso contrário, a nota final será zero.
